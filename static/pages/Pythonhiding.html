Data Hiding in Python

What is Data Hiding?

Data hiding is the concept of restricting direct access to certain variables and methods of a class. It protects the internal data of an object and prevents accidental modification.

Private Variables

In Python, private variables are created using a double underscore (__) before the variable name. 


class Student:

    def __init__(self):
        self.__marks = 90

s = Student()

print(s.__marks)
AttributeError: 'Student' object has no attribute '__marks'

Name Mangling

Python does not completely hide private variables. Instead, it changes the variable name internally. This mechanism is called Name Mangling. 


class Student:

    def __init__(self):
        self.__marks = 90

s = Student()

print(s._Student__marks)
90

Python internally converts __marks to _Student__marks.

Getter and Setter Methods

Getter and setter methods are used to safely access and modify private variables. 


class Student:

    def __init__(self):
        self.__marks = 0

    def getMarks(self):
        return self.__marks

    def setMarks(self, m):
        self.__marks = m

s = Student()

s.setMarks(85)
print(s.getMarks())
85

Advantages of Data Hiding

✔ Protects internal data of objects
✔ Prevents accidental modification
✔ Improves program security
✔ Supports encapsulation
